Stage 3 — Pre-Contract QA

Two-pass check that the final contract pack reflects the signed source before issue. Pass 1 compares pricing & specification; Pass 2 is an AI-assisted drawing review that points the reviewer to sheets needing human confirmation. Output is itemised by spec section and drawing page, matching the QA template. All three contract documents plus at least one signed source are required — otherwise the job is parked, not passed.

Rule Library

Learning Panel — Recent False Positives

Issues staff marked as false positives or not-applicable across Stage 2/3. Use these to decide whether to add an exclusion to a rule (so the tool stops flagging them) or lower a rule's severity. This is how the tool gets more accurate over time — no developer needed.

Access Code Management

Generate codes for each consultant. Each code is unique and tied to their name. Deactivate to revoke access.

User Management

Add users and assign roles. Each user logs in with their own code. Roles: Admin (everything), Manager (Heath) (consultant view + Rules + Learning + Heath Dashboard + Plans), Manager (Lyana) (consultant view + Rules + Learning), Consultant (submissions, own history, Help).

Feature Flags
